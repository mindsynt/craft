# Craft — 团队协作工作流

## 团队架构

```
👤 架构师/主管 ─── 我
  ├── 🎨 UI设计师 (1人)
  ├── 💻 全栈开发 (3人)
  ├── 👀 Code Review (2人)
  ├── 🧪 测试 (3人)
  ├── ⚙️ 运维 (1人)
  └── 🛠️ 打杂 (1人) ── 给架构师打下手
```

## 工作流

```
你 → 需求 → 我(架构师) → 拆解 → Kanban → 分配 → 执行 → Review → 测试 → 完成
                      ↑                                              │
                      └────────────── 你确认完成 ←────────────────────┘
```

## 各角色职责

### 👤 架构师/主管（我）
- 接收你的指令
- **一般情况不下场干活**
- 需求讨论、架构设计、任务拆分
- 贴任务到 Kanban，分配给人
- 追踪进度，汇报

### 🎨 UI设计师
- 终端界面布局设计（Textual TUI）
- 主题设计、配色方案
- 用户交互流程设计
- 输出: 界面原型 / CSS 样式 / 交互文档

### 💻 全栈开发（3人）
- Python 后端开发（FastAPI/core 模块）
- 前端/终端 UI 开发（Textual/TUI）
- 功能实现、接口开发
- 输出: 可运行的代码 / PR

### 👀 Code Review（2人）
- 审查代码质量、架构一致性
- 检查是否与 MiMo-Code 架构对齐
- 输出: Review 意见 / 批准/驳回

### 🧪 测试（3人）
- 单元测试、集成测试、回归测试
- 用 `audit.py` 和 `monitor.py` 验证
- 输出: 测试报告 / Bug 列表

### ⚙️ 运维（1人）
- 环境配置、依赖管理
- CI/CD 流程
- 部署、发布
- 输出: 部署脚本 / 环境文档

### 🛠️ 打杂（1人）
- 给架构师打下手
- 文档编写、日志整理
- 简单任务执行
- 输出: 文档 / 整理结果

## Kanban 状态流转

```
Backlog → To Do → In Progress → Review → Testing → Done
   ↑                                        │
   └──────────── Failed ←───────────────────┘
```

## 通讯协议

```
你 → "做个XX功能"
我 → 需求讨论 → 拆解任务 → 贴到 Kanban
  → "🎨 UI设计师 设计XX界面"
  → "💻 开发1 实现XX逻辑"
  → "👀 Review1 审查XX"
  → "🧪 测试1 测试XX"
  → "你确认一下效果"
```
